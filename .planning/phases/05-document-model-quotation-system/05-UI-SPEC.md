### Status Badge Colors

Status badges use the existing `Badge` component with `className` overrides for semantic colors. Each status maps to a specific background + text combination:

| Status | Thai Label | Background Class | Text Class |
|--------|-----------|-----------------|-----------|
| draft | แบบร่าง | `bg-secondary` | `text-secondary-foreground` |
| sent | ส่งแล้ว | `bg-blue-100 dark:bg-blue-900/30` | `text-blue-700 dark:text-blue-300` |
| accepted | อนุมัติ | `bg-green-100 dark:bg-green-900/30` | `text-green-700 dark:text-green-300` |
| rejected | ปฏิเสธ | `bg-destructive/10` | `text-destructive` |
| expired | หมดอายุ | `bg-orange-100 dark:bg-orange-900/30` | `text-orange-700 dark:text-orange-300` |
| converted | แปลงแล้ว | `bg-purple-100 dark:bg-purple-900/30` | `text-purple-700 dark:text-purple-300` |
| voided | ยกเลิก | `bg-muted` | `text-muted-foreground` |

**Rationale:** The existing Badge component only has `default`, `secondary`, `destructive`, and `outline` variants. Status colors are applied via className override on the base Badge, using Tailwind's built-in color palette for blue/green/orange/purple (no custom CSS variables needed). Dark mode uses the `/30` opacity pattern for subtle backgrounds.

**Source:** `components/ui/badge.tsx` (existing variants), `05-RESEARCH.md` (QUOTATION_STATUSES from competitor analysis)

## Interaction Contracts

### Line Item Management

| Interaction | Behavior |
|------------|----------|
| "+ เพิ่มรายการ" click | Append new empty row: `{ description: "", quantity: "1", unit: "ชิ้น", unitPrice: "", discount: "0" }`. Focus on the new row's description input. |
| Trash icon click | Remove row if items.length > 1. If only 1 row remains, button is `disabled`. No confirmation needed. The delete button MUST use `<Tooltip><TooltipTrigger asChild><Button variant="ghost" size="icon" aria-label="ลบรายการ">...</Button></TooltipTrigger><TooltipContent>ลบรายการ</TooltipContent></Tooltip>`. |
| Amount column | Auto-calculated: `(qty * unitPrice) - discount`. Read-only display using `formatCurrency()`. |
| Quantity/price/discount change | Totals section recalculates via `useMemo` on every keystroke (same pattern as tax-invoice). |

### VAT Toggle

| Interaction | Behavior |
|------------|----------|
| Switch ON | Show "+7% VAT" row in totals. VAT computed via `computeVATOnSubtotal(subtotalAfterDiscount)`. Total = subtotalAfterDiscount + vatAmount. |
| Switch OFF | VAT row hidden. Total = subtotalAfterDiscount. `vatRate` stored as 0 on save. |
| Default state | OFF (quotations may or may not include VAT per D-08). |

### Status Transitions

| Current Status | Available Actions | Button Variant | Confirmation |
|---------------|-------------------|----------------|--------------|
| draft | "ส่งใบเสนอราคา" (to sent) | `default` | None |
| draft | "ยกเลิก" (to voided) | `destructive` | Dialog confirmation |
| sent | "ลูกค้าอนุมัติ" (to accepted) | `default` | None |
| sent | "ลูกค้าปฏิเสธ" (to rejected) | `outline` | None |
| sent | "ยกเลิก" (to voided) | `destructive` | Dialog confirmation |
| accepted | No actions in Phase 5 (convert to invoice is Phase 6) | -- | -- |
| rejected | Terminal state -- no actions | -- | -- |
| expired | Terminal state -- no actions (computed at query time) | -- | -- |
| converted | Terminal state -- no actions | -- | -- |
| voided | Terminal state -- no actions | -- | -- |

**Edit rule:** Only `draft` status quotations can be edited. All other statuses show read-only detail view.

### PDF Download

| Interaction | Behavior |
|------------|----------|
| Download button click | Show `Loader2` spinner + "กำลังสร้าง PDF..." text. Call `pdf(createElement(QuotationPDF, { data })).toBlob()`. Download as `{documentNumber}.pdf`. Toast success: "PDF พร้อมดาวน์โหลด". |
| Download error | Toast error: "สร้าง PDF ไม่สำเร็จ -- กรุณาลองใหม่อีกครั้ง". Log to console. |

### Contact Selection

| Interaction | Behavior |
|------------|----------|
| Search | Existing `ContactAutocomplete` -- type to search, select from dropdown |
| Create new | "สร้างผู้ติดต่อใหม่" opens `ContactInlineCreate` dialog |
| Selection display | Show contact info in `bg-muted/50 p-3 rounded-md` box: name, Tax ID, branch, address |

### Line Item Delete Button Accessibility

The trash icon button in each line item row MUST include both `aria-label` and a wrapping `Tooltip`:

```tsx
<TooltipProvider>
  <Tooltip>
    <TooltipTrigger asChild>
      <Button
        variant="ghost"
        size="icon"
        aria-label="ลบรายการ"
        disabled={items.length <= 1}
        onClick={() => removeItem(index)}
      >
        <Trash2 className="h-4 w-4" />
```
