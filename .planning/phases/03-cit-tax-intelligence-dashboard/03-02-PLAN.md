    **4. actions.ts (per D-07):**

    `createCreditNoteAction(prevState, formData)`:
    - Parse formData with creditNoteFormSchema.safeParse
    - Get current user, business profile
    - Load original invoice from AppData (key = `tax-invoice-{originalInvoiceKey}`)
    - If not found, return error "ไม่พบใบกำกับภาษีต้นฉบับ"
    - Generate sequential document number: CN-YYYY-NNNN for credit, DN-YYYY-NNNN for debit (reuse Setting model atomic counter pattern from tax-invoice, prefix "seq_credit_note" or "seq_debit_note")
    - Compute: totalDifference = sum of (correctedAmount - originalAmount) per item. For credit note: difference is negative (decrease). For debit note: positive (increase).
    - Compute VAT adjustment using extractVATFromTotal on the absolute difference
    - Store in AppData: key = `credit-note-{docNumber}`, JSON with all fields + original invoice ref + business profile snapshot
    - Create adjustment Transaction (per D-07 "auto-adjusts linked transaction amounts"):
      - For credit note: type="income" with negative total (reduces revenue)
      - For debit note: type="income" with positive total (increases revenue)
      - Set vatType="output", vatAmount accordingly, documentNumber=docNumber
    - Return { success: true, data: { docNumber, noteData } }

    `createDebitNoteAction(prevState, formData)`: call createCreditNoteAction with noteType override to "debit"

    **5. credit-note-form.tsx:**
    Client component. Clone tax-invoice-form pattern.
    - noteType toggle: ใบลดหนี้ (credit) / ใบเพิ่มหนี้ (debit)
    - Original invoice selector: text input for invoice number (searches AppData keys starting with "tax-invoice-")
    - Date picker for issuedAt
    - Reason text input
    - Dynamic line items array (add/remove rows): description, original amount (baht), corrected amount (baht). Show difference per row.
    - Total section: original total, corrected total, difference, VAT on difference
    - Submit button, on success show NotePreview dialog

    **6. credit-note-pdf.tsx:**

    - Seller info from business profile (name, Tax ID, branch, address)
    - Buyer info from original invoice
    - Reference to original invoice: number, date
    - Reason for issuance
    - Items table: description, original amount, corrected amount, difference
    - Total: difference amount, VAT on difference
    - THSarabunNew font, same styles as tax-invoice-pdf

    **7. note-preview.tsx:**
    Preview dialog with PDF download. Clone tax-invoice preview pattern (Dialog with PDFDownloadLink).
