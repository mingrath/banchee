## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 14px | 400 (regular) | 1.5 | Noto Sans Thai |
| Label | 14px | 700 (bold) | 1.4 | Noto Sans Thai |
| Heading | 20px | 700 (bold) | 1.2 | Noto Sans Thai |
| Display | 24px | 700 (bold) | 1.2 | Noto Sans Thai |

### Thai-Specific Typography Notes

- Noto Sans Thai requires `subsets: ["thai", "latin"]` in next/font config.
- Load weights 400, 500, 600, 700 via `next/font/google`. Use CSS variable `--font-noto-sans-thai`.
- Thai text has taller vertical metrics than Latin. Line-height 1.5 on body text prevents tone mark clipping.
- Tax amounts on dashboard cards use Display size (24px bold) -- matches existing `text-2xl font-bold` pattern in stats-widget.tsx.
- Form labels for Thai tax terms (e.g. "เลขประจำตัวผู้เสียภาษี") are Label size (14px medium).
- Revenue Department terminology must remain in Thai only -- no English translations in primary UI (D-04, RESEARCH.md anti-pattern).

## Interaction Contracts

### Setup Wizard (D-14)

| Step | Fields | Validation |
|------|--------|-----------|
| 1. Company Name | `biz_company_name` (text) | Required. Min 2 chars. |
| 2. Tax ID | `biz_tax_id` (text, 13-digit mask) | Required. Exactly 13 digits. Regex: `/^\d{13}$/` |
| 3. Branch | `biz_branch` (select: "สำนักงานใหญ่" = "00000", or text input for branch number) | Required. Default: "00000" (HQ). |
| 4. Address | `biz_address` (textarea) | Required. |
| 5. VAT Registration | `biz_vat_registered` (switch toggle) | Boolean. Default: false. If true, show VAT registration date picker. |
| 6. Accounting Period | `biz_fiscal_year_start` (select: months 1-12) | Default: 1 (January = calendar year). |
| 7. LLM API Key | Provider selector + API key input | Required. At least one provider configured. Test connection on submit. |

**Navigation:** Back/Next buttons at bottom. "Next" disabled until current step validates. Final step shows "เริ่มใช้งาน" (Start Using) button.

**Transitions:** Simple slide-left on forward, slide-right on back. Duration 200ms ease-in-out. Use Claude's discretion.

**Gate behavior:** Redirect to `/setup` from any authenticated route until all required steps are complete. Store completion flag via `isBusinessProfileComplete()`.

### AI Receipt Scanning with Thai Validation (D-06, D-07)

**Flow:**
1. User uploads receipt image (existing drop zone).
2. User clicks "วิเคราะห์" (Analyze) button.
3. **Loading state:** Skeleton pulse on form fields. Analyze button shows Loader2 spinner + "กำลังวิเคราะห์..." text. Duration: typically 3-8 seconds.
4. AI fills all fields. Each field shows inline validation:
   - Green checkmark badge: field present and valid
   - Red cross badge: field missing or invalid
   - Gray dash badge: field not applicable (e.g., branch for non-VAT receipt)
5. User reviews and corrects fields.
6. User clicks "บันทึก" (Save) to confirm.

**Validation badge placement:** Right-aligned inside the form field row, after the input. Use Badge component with variant `outline` and colored dot prefix.

### VAT Dashboard (D-11)

**Layout:** Below existing income/expense/profit/transactions card grid. Separate section with heading "ภาษีมูลค่าเพิ่ม (VAT)".

**Cards (3-column grid on desktop, stack on mobile):**

| Card | Title | Value Format | Color |
|------|-------|-------------|-------|
| ภาษีขาย (Output VAT) | "ภาษีขาย" | "฿X,XXX.XX" | Green gradient |
| ภาษีซื้อ (Input VAT) | "ภาษีซื้อ" | "฿X,XXX.XX" | Blue gradient |
| ภาษีสุทธิ (Net VAT) | "ต้องชำระ" or "เครดิตภาษี" | "฿X,XXX.XX" | Amber (payable) or Green (credit) |

**Link:** Each card links to VAT report page. Net VAT card shows "ดูรายละเอียด" (View details) text link.

**Conditional display:** VAT section hidden when `biz_vat_registered = false` (D-15, BIZ-02).

### PP30 Report Generation (D-12)

**Flow:**
1. User navigates to VAT report page (via dashboard "ดูรายละเอียด" link or sidebar).
2. Month/year selector at top (default: previous month).
3. "ออกรายงาน" (Generate Report) button -- single click generates all 3 documents.
4. **Loading state:** Button shows Loader2 spinner + "กำลังสร้างรายงาน..."
5. Preview dialog opens showing summary: Output VAT total, Input VAT total, Net payable/credit.
6. Three download buttons: "รายงานภาษีซื้อ", "รายงานภาษีขาย", "ภ.พ.30"
7. "ดาวน์โหลดทั้งหมด" (Download All) button generates ZIP with all 3 PDFs.

**PP30 Preview Summary Fields (displayed in Card):**

| Row | Label | Value |
|-----|-------|-------|
| 1 | ยอดขายในเดือนนี้ | ฿X,XXX.XX |
| 2 | ภาษีขายเดือนนี้ | ฿X,XXX.XX |
| 3 | ยอดซื้อในเดือนนี้ | ฿X,XXX.XX |
| 4 | ภาษีซื้อเดือนนี้ | ฿X,XXX.XX |
| 5 | ภาษีที่ต้องชำระ / เครดิตภาษี | ฿X,XXX.XX (color-coded) |

### VAT Expiry Warning (D-13, VAT-07)

**Trigger:** Input VAT invoices where `issuedAt` is within 30 days of 6-month mark.

**Display:** Alert component below VAT summary cards. AlertTriangle icon (Lucide). Amber background.

**Copy:** "[N] ใบกำกับภาษีใกล้หมดอายุ (6 เดือน)" with expandable list showing vendor name + invoice date + days remaining.

### Revenue Threshold Alert (VAT-03)

**Trigger:** Cumulative income YTD >= 80% of 1,800,000 THB (i.e., >= 1,440,000 THB).

**Display:** Alert component at top of dashboard. Info variant (blue).

**Copy:** "รายได้สะสมปีนี้ ฿X,XXX,XXX -- ใกล้ถึงเกณฑ์จดทะเบียน VAT (฿1,800,000)" with link to settings to enable VAT registration.
