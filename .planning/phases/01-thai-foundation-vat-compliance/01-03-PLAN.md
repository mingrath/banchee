    Per RESEARCH.md Pattern 2 and CONTEXT.md D-06. Create the validators directory and module:

    ```typescript
    export type FieldValidationStatus = "valid" | "invalid" | "missing" | "not_applicable"

    export type FieldValidation = {
      status: FieldValidationStatus
      value: string | null
      message?: string
    }

    export type ValidationResult = {
      isValidTaxInvoice: boolean
      fields: Record<string, FieldValidation>
      missingCount: number
      warnings: string[]
    }

    export function validateTaxInvoiceFields(
      extractedData: Record<string, unknown>
    ): ValidationResult
    ```

    Validation checks per Section 86/4 (THAI_TAX_REFERENCE.md):
    1. `tax_invoice_label`: Check if "ใบกำกับภาษี" text was detected (from name or description field)
    2. `seller_name`: Check merchant field is non-empty
    3. `seller_tax_id`: Check merchant_tax_id matches /^\d{13}$/
    4. `seller_address`: Check if address info present (from description or dedicated field)
    5. `seller_branch`: Check merchant_branch is present ("00000" or branch number)
    6. `buyer_info`: Check if buyer fields present (may be N/A for simplified invoices)
    7. `invoice_number`: Check document_number is non-empty
    8. `items_and_values`: Check items array is non-empty with at least one item
    9. `vat_separated`: Check vat_amount is present and > 0
    10. `issue_date`: Check issuedAt is present and valid date
    11. `buyer_tax_id`: Check if present (for B2B -- set to not_applicable if not found, as it is only required for B2B)

    Return per-field status for inline display. `isValidTaxInvoice` = true only if ALL required fields (1-10) are valid.

    Also add B.E. date correction:
    ```typescript
    export function correctBuddhistEraDate(dateStr: string): string
    // If year > 2500, subtract 543. Returns ISO date string.
    // Handles formats: "23/03/2569", "2569-03-23", "23 มี.ค. 2569"
    ```
