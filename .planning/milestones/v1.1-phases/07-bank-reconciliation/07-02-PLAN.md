    Create `app/(app)/apps/bank-reconciliation/actions.ts` with `"use server"` directive. All actions follow the established pattern: `getCurrentUser()` for auth, try/catch, return `ActionState<T>`, `revalidatePath("/apps/bank-reconciliation")`.

    **importBankStatementAction(prevState, formData)** per D-01, D-02, D-04, D-05, D-06, D-11:
    1. Extract file from formData (`.get("file") as File`)
    2. Read file buffer (`Buffer.from(await file.arrayBuffer())`)
    3. Determine file type from filename extension (.csv or .xlsx)
    4. Parse: CSV via `parseCSVBuffer(buffer, skipLines)`, Excel via `parseExcelBuffer(buffer)`
    5. Extract column mapping from formData (date, description, deposit, withdrawal, balance, reference as column indices). Validate with `columnMappingSchema`.
    6. Extract bankName from formData. Validate with `importBankStatementSchema`.
    7. Look up bank preset for `useBuddhistEra` flag. For "other", default to false.
    8. Call `parseBankEntries(rows, mapping, useBuddhistEra)` to get `ParsedBankEntry[]`
    9. Generate file hash via `generateFileHash(buffer)`. Check for duplicate via `findStatementByHash`.
    10. Create BankStatement via `createBankStatement(userId, { bankName, filename, fileHash, totalEntries })`.
    11. Create BankEntry rows via `createBankEntries(statementId, parsedEntries)`.
    12. Run auto-matching: fetch all user transactions, call `findMatches` for each entry, update entries with `matchStatus: "suggested"`, `matchScore`, `matchReasons`, `transactionId` for those with score >= 60.
    13. Update statement status via `updateStatementStatus`.
    14. Return `{ success: true, data: { statementId } }`.
    15. Hard limit: reject files with > 10000 data rows with error "ไฟล์มีข้อมูลมากเกินไป (สูงสุด 10,000 รายการ)".

    **confirmMatchAction(entryId: string, transactionId: string)** per D-08:
    1. Update entry: matchStatus = "confirmed", transactionId set.
    2. Call `updateStatementStatus` to recompute statement status.
    3. Revalidate path.

    **rejectMatchAction(entryId: string):**
    1. Update entry: matchStatus = "unmatched", transactionId = null, matchScore = null, matchReasons = null.
    2. Call `updateStatementStatus`.

    **createTransactionFromEntryAction(entryId: string)** per D-10, BANK-05:
    1. Read the BankEntry to get amount, date, description.
    2. Determine transaction type: if deposit > 0 then "income", else "expense".
    3. Determine amount: `deposit || withdrawal` (whichever is non-zero), already in satang.
    4. Call `createTransaction(userId, { name: entry.description, total: amount, type, issuedAt: entry.date })`.
    5. Update entry: matchStatus = "created", transactionId = new transaction id.
    6. Call `updateStatementStatus`.

    **skipEntryAction(entryId: string):**
    1. Update entry: matchStatus = "skipped".
    2. Call `updateStatementStatus`.

    **undoMatchAction(entryId: string):**
    1. Update entry: matchStatus = "unmatched", transactionId = null.
    2. Call `updateStatementStatus`.

    **deleteStatementAction(statementId: string):**
    1. Call `deleteBankStatement(statementId)` (cascades to entries).
    2. Revalidate path.

    **CRITICAL:** Every action calls `getCurrentUser()` first. Every action wraps in try/catch. Every action returns `ActionState<T>`. Every action calls `revalidatePath("/apps/bank-reconciliation")`.
