    Functions:
    - `scoreMatch(entryAmount, entryDate, entryDescription, txTotal, txDate, txName)` per D-07:
      - Amount: 40 points for `Math.abs(entryAmount) === Math.abs(txTotal)` (absolute values per Pitfall 6)
      - Date: 30 points same day, 25 for 1 day, 20 for 2-3 days, 10 for 4-7 days, 0 for >7 days. Uses `differenceInDays` from date-fns.

    - `findMatches(entry: {amount, date, description}, transactions: {id, total, issuedAt, name}[], threshold?: number)`:
      - Score each transaction against the entry
      - Filter to score >= threshold (default 60 per D-07)
      - Sort descending by score
      - Return MatchCandidate[]

    Import `differenceInDays` from date-fns. No other external deps.

    **CRITICAL per D-07:** Threshold is 60%. Weights are amount=40, date=30, description=30.
    **CRITICAL:** Never auto-confirm matches. This service only scores and suggests.
