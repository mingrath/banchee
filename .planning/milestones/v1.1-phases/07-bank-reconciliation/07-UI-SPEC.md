## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline padding, badge padding |
| sm | 8px | Compact element spacing (`gap-2`, `space-y-2`), column mapping dropdown gap |
| md | 16px | Default element spacing (`space-y-4`, `gap-4`), Card content padding |
| lg | 24px | Section padding (`space-y-6`), Card section gaps, review table row padding |
| xl | 32px | Page header bottom margin (`mb-8`), statement list top margin |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: Upload drop zone has 48px vertical padding (`p-12`) for visual breathing room -- matches existing `drop-zone-widget.tsx` pattern.

**Source:** Measured from `drop-zone-widget.tsx` (p-8), `document-table.tsx` (space-y-4 between sections), Phase 5 UI-SPEC spacing scale.

## Typography

### UI Typography (Noto Sans Thai)

Two weights only: **500 (medium)** for all body/label/header text and **700 (bold)** for the page heading.

| Role | Size | Weight | Line Height | Tailwind Classes |
|------|------|--------|-------------|-----------------|
| Page heading | 30px (1.875rem) | 700 (bold) | 1.2 | `text-3xl font-bold tracking-tight` |
| Card title | 18px (1.125rem) | 500 (medium) | 1.4 | `text-lg font-medium` (CardTitle component) |
| Body / form labels | 14px (0.875rem) | 500 (medium) | 1.5 | `text-sm font-medium` |
| Table header | 14px (0.875rem) | 500 (medium) | 1.5 | `text-sm font-medium text-muted-foreground` |
| Preview data (column mapping) | 14px (0.875rem) | 500 (medium) | 1.5 | `text-sm font-medium text-muted-foreground` |
| Match score | 12px (0.75rem) | 500 (medium) | 1.0 | `text-xs font-medium` |
| Status badge | 12px (0.75rem) | 500 (medium) | 1.0 | `text-xs font-medium` (Badge component) |
| Progress label | 14px (0.875rem) | 500 (medium) | 1.5 | `text-sm font-medium` |

Note: Preview data and table headers share the same weight (500 medium). Preview data is visually de-emphasized via `text-muted-foreground` color, not a separate font weight.

**Source:** Phase 5 UI-SPEC typography, `document-table.tsx` table header classes

## Interaction Contracts

### File Upload (StatementUpload)

| Interaction | Behavior |
|------------|----------|
| Click drop zone | Opens native file picker filtered to `.csv,.xlsx` (`accept=".csv,.xlsx,application/vnd.openxmlformats-officedocument.spreadsheetml.sheet"`) |
| Drag file over zone | Zone border changes to `border-primary`, background to `bg-primary/5`. Text changes to "วางไฟล์ที่นี่" |
| Drag leave zone | Reset to default dashed border and text |
| Drop file on zone | Parse file buffer, show filename below zone, reveal bank preset selector |
| Drop invalid file type | Toast error: "รองรับเฉพาะไฟล์ CSV และ Excel (.xlsx)" |
| File selected via click | Same behavior as drop |

### Bank Preset Selector

| Interaction | Behavior |
|------------|----------|
| Select "กสิกรไทย" (KBank) | Pre-populate column mapping dropdowns with KBank default positions. Show encoding notice: "ตรวจพบการเข้ารหัส TIS-620 -- แปลงเป็น UTF-8 อัตโนมัติ" (if TIS-620 detected) |
| Select "ไทยพาณิชย์" (SCB) | Pre-populate column mapping dropdowns with SCB defaults |
| Select "กรุงเทพ" (BBL) | Pre-populate column mapping dropdowns with BBL defaults |
| Select "อื่นๆ" (Other) | All column mapping dropdowns reset to "Skip" -- user must map manually |
| Change selection | Re-apply preset mapping, overwriting current selections. Preview rows remain unchanged. |

### Column Mapper (ColumnMapper)

| Interaction | Behavior |
|------------|----------|
| Select dropdown change | Update column mapping for that column index. Each field (Date, Description, Deposit, Withdrawal, Balance, Reference) can only be assigned to one column -- selecting it elsewhere auto-sets the previous column to "Skip". |
| "Skip" selection | Column excluded from import. All preview cells in that column show dimmed text (`text-muted-foreground/50`). |
| Required fields missing | Import button disabled. Inline validation below table: "กรุณาเลือกคอลัมน์สำหรับ วันที่, รายละเอียด, และ ฝาก หรือ ถอน อย่างน้อย 1 คอลัมน์" |
| Preview row display | First 3 data rows from the parsed file. Read-only. `bg-muted/50 rounded text-sm`. If file has fewer than 3 data rows, show all available rows. |

### Match Review (MatchReviewTable)

| Interaction | Behavior |
|------------|----------|
| Confirm button (Check icon) | Calls `confirmMatchAction(entryId, transactionId)`. Row transitions to confirmed state: green tint background, confirm button becomes disabled, shows "ยืนยันแล้ว" badge. Progress bar updates. |
| Reject button (X icon) | Calls `rejectMatchAction(entryId)`. Suggested match removed. Row becomes "unmatched" with "สร้างรายการ" and "ข้าม" buttons visible. |
| "สร้างรายการ" button (Plus icon) | Calls `createTransactionFromEntryAction(entryId)`. Pre-fills transaction with entry amount (deposit as income, withdrawal as expense), date, and description. Row transitions to "created" state with purple badge. Progress bar updates. |
| "ข้าม" button (SkipForward icon) | Calls `skipEntryAction(entryId)`. Row grayed out with "ข้าม" badge. Progress bar updates. |
| "ดูตัวเลือกอื่น" expand | Collapsible opens to show alternative match candidates sorted by score descending. Each alternative shows: transaction name, date, amount, score. "สลับ" (Switch) button next to each alternative. |
| "สลับ" (Switch) button in alternatives | Replaces current suggested match with the selected alternative. Closes collapsible. Row updates to show new matched transaction. |
| Undo confirmed match | Shows "ยกเลิก" button (variant="ghost", Undo2 icon) on confirmed rows. Calls `undoMatchAction(entryId)`. Row reverts to "unmatched". Progress bar decrements. |
| Row click (unmatched, no suggestion) | Opens manual match dialog: Select dropdown listing all unmatched BanChee transactions, filtered by amount proximity. Selecting a transaction and clicking "จับคู่" confirms the manual match. |

### Statement List (StatementList)

| Interaction | Behavior |
|------------|----------|
| Row click | Navigate to `/apps/bank-reconciliation/[statementId]` review page |
| Progress column | Shows `Progress` component (h-2, inline) + text "45/50" to the right. Progress value = `(matchedEntries / totalEntries) * 100`. |
| Delete statement | Context menu or trailing icon button (Trash2, variant="ghost"). Opens Dialog confirmation. Deletes statement and all entries. |

### Reconciliation Progress (ReconciliationProgress)

| Interaction | Behavior |
|------------|----------|
| Auto-update | Re-renders after each confirm/reject/create/skip/undo action via `revalidatePath`. |
| 100% completion | Progress bar fills to 100%. Toast: "กระทบยอดเสร็จสิ้น" Statement status auto-updates to "reconciled". StatusBadge changes to green "กระทบยอดเสร็จ". |
| Label format | "{resolved}/{total} รายการ" where resolved = confirmed + created + skipped entries. |

## Accessibility

| Element | Requirement |
|---------|------------|
| Drop zone | `role="button"`, `tabIndex={0}`, `aria-label="อัปโหลดไฟล์ CSV หรือ Excel"`, responds to Enter/Space key press |
| Column mapping dropdowns | Each Select has `aria-label` matching the column index: "เลือกประเภทข้อมูลสำหรับคอลัมน์ {n}" |
| Confirm/reject buttons | Icon-only buttons MUST use `aria-label` + `Tooltip` wrapping (same pattern as Phase 5 line item delete) |
| Progress bar | Uses Radix `Progress` primitive which provides `role="progressbar"`, `aria-valuenow`, `aria-valuemax` automatically |
| Collapsible alternatives | `CollapsibleTrigger` receives `aria-expanded` from Radix automatically |
| Amount colors | Green/red amount coloring is supplemented by +/- sign prefix, not color-only: "+5,000.00" (green) vs "-1,500.00" (red) |
| Table row navigation | Statement list rows are clickable via `onClick` + `cursor-pointer`. Also provide a visually hidden link for keyboard navigation. |
